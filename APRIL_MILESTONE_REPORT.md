## 1) PKI Integration (Two Security-Critical Functions)

### A. Resume Integrity Signing and Verification
- On resume upload, backend computes SHA-256 hash and signs it with server RSA private key.
- On download/integrity-check, backend verifies hash and PKI signature before returning success.
- New integrity endpoint exposes verification state and signer public key.

### B. Audit Verification Snapshot Signing
- Audit-chain verification output is now signed using server RSA private key.
- Signature and public key are returned with `/admin/audit-logs/verify`.
- This proves the audit verification snapshot was generated by trusted server identity.

## 2) OTP with Virtual Keyboard for High-Risk Actions

High-risk actions now require OTP entered through a virtual numeric keypad in the UI.

Implemented actions:
- **Resume download** (`resume_download` OTP purpose)
- **Resume delete** (`resume_delete` OTP purpose)
- **Password reset confirmation** (OTP entered through dedicated Forgot Password virtual keypad page)

Flow:
1. User initiates protected action.
2. User requests OTP (`/auth/high-risk-otp/request`) for resume actions, or password-reset OTP via `/auth/password-reset`.
3. User enters OTP using on-screen virtual keyboard.
4. Backend validates OTP purpose/context before executing the action.

## 4) Demonstrated Defenses Against Common Web Attacks

### SQL Injection
- SQLAlchemy ORM parameterized queries are used instead of raw string concatenation.
- Invalid/malicious values are not interpolated into SQL directly.

### XSS
- React default escaping prevents direct script injection in rendered text.
- Security headers and CSP are added from backend middleware.

### CSRF
- API uses bearer tokens in Authorization headers (not cookie-bound auth flow), reducing CSRF attack surface.
- CORS origin controls restrict browser cross-origin credentialed requests.

### Session Fixation / Hijacking
- JWT access/refresh token flow with expiry and refresh path.
- Suspended/inactive users are blocked at dependency layer.
- Trusted host validation and security response headers added.
